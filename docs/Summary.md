# Отчёт по итогам автоматизации

**Запланировано:**
- Автоматизировать тестирование формы оплаты.
- Покрыть позитивные и негативные сценарии.
- Настроить Allure для генерации отчётов.
- Обеспечить корректную работу ожиданий и валидаций.
- Подготовить итоговый отчёт по автоматизации.

**Реализовано:**
- Написано 30 UI‑тестов на Selenide + JUnit5.
- Покрыты все поля формы: номер карты, месяц, год, владелец, CVC.
- Реализованы позитивные и негативные сценарии.
- Интегрирован Allure + Allure‑Selenide.
- Добавлен AllureListener для шагов, скриншотов и HTML.
- Настроены ожидания до 15 секунд.
- Сгенерирован Allure‑отчёт (12 упавших, 18 успешных тестов).

## Причины, по которым что-то не было реализовано

- Часть тестов падает из‑за некорректной валидации на стороне приложения.
- Некоторые негативные сценарии требуют доработки бэкенда, чтобы корректно возвращать ошибки.

## Сработавшие риски

- Проблемы с интеграцией AllureListener и JUnit Extension — устранено.
- IntelliJ не видел тестовые классы из‑за неверной индексации — исправлено.
- WebDriver создавался с задержкой, что влияло на стабильность тестов.
- Allure‑отчёт не генерировался при запуске тестов через IntelliJ — решено запуском через Gradle.

## Общий итог по времени

**Запланировано:** ~6 часов  
**Фактически:** ~10 часов

**Причины перерасхода:**
- 2 часа ушло на устранение проблем с AllureListener и JUnit Extension.
- 1.5 часа — настройка Gradle и кодировки UTF‑8.
- 1 час — отладка нестабильных тестов.
- 0.5 часа — генерация отчёта и проверка скриншотов.
