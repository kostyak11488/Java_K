# Отчёт о тестировании

## Цель тестирования

Проверить корректность работы веб‑сервиса «Путешествие дня» при оплате тура банковской картой, включая:
- обработку валидных и невалидных данных;
- взаимодействие с тестовым банковским сервисом;
- корректность записей в базе данных PostgreSQL.

---

## Тестовое окружение

- ОС: Windows 10 / Windows 11
- Java 11 (JDK 11)
- Docker + Docker Compose
- PostgreSQL 13 (контейнер postgres-db)
- Браузер: Google Chrome
- Тестируемый сервис: aqa-shop.jar
- Инструменты: Selenide, JUnit 5, Allure

---

## Запуск окружения

1. Запуск PostgreSQL  
   docker-compose up -d

2. Запуск тестируемого сервиса  
   java -jar aqa-shop.jar

3. Запуск автотестов  
   ./gradlew clean test

4. Формирование отчёта  
   ./gradlew allureReport  
   ./gradlew allureServe

---

## Результаты тестирования

### Успешные тесты

- Оплата тура валидной картой (APPROVED)
- Отказ в оплате при использовании отклонённой карты (DECLINED)
- Проверка записи статуса операции в таблицу payment_entity
- Проверка валидации полей формы (месяц, год, владелец, CVC)
- Проверка отображения сообщений об ошибках при невалидных данных

### Неуспешные тесты

- Обнаружены несоответствия в валидации отдельных полей формы
- Все выявленные дефекты оформлены в bug-report.md

---

## Общий вывод

Основной функционал оплаты тура работает корректно при использовании валидных данных.  
Система корректно обрабатывает отклонённые карты и отображает соответствующие статусы.  
Найденные дефекты не блокируют ключевой сценарий оплаты и могут быть исправлены в плановом порядке.
