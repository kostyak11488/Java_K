# Отчёт о дефектах

## Формат описания дефектов

Каждый дефект содержит:
- ID
- Название
- Описание
- Шаги воспроизведения
- Ожидаемый результат
- Фактический результат
- Окружение

---

## Дефект 1

**ID:** BUG-001  
**Название:** Поле «Владелец» принимает кириллицу  
**Описание:** Поле должно принимать только латинские буквы, но допускает ввод кириллицы.

**Шаги воспроизведения:**
1. Открыть форму оплаты
2. Ввести в поле «Владелец» значение «ИВАН ИВАНОВ»
3. Нажать «Продолжить»

**Ожидаемый результат:**  
Отображается сообщение об ошибке: «Неверный формат имени владельца».

**Фактический результат:**  
Форма принимает значение, ошибка не отображается.

**Окружение:**  
Windows 10, Chrome, Java 11, PostgreSQL 13.

---

## Дефект 2

**ID:** BUG-002  
**Название:** Поле CVC принимает значение «000»  
**Описание:** Значение «000» считается невалидным, но система принимает его.

**Шаги воспроизведения:**
1. Открыть форму оплаты
2. Ввести CVC = 000
3. Нажать «Продолжить»

**Ожидаемый результат:**  
Ошибка: «Неверный CVC».

**Фактический результат:**  
Форма принимает значение, ошибка отсутствует.

**Окружение:**  
Windows 11, Chrome, Java 11, PostgreSQL 13.

## Дефект 3

**ID:** BUG-003  
**Название:** Поле «Месяц» принимает значение 00  
**Описание:** Значение «00» не является корректным месяцем, но система не блокирует ввод.

**Шаги воспроизведения:**
1. Ввести месяц = 00
2. Нажать «Продолжить»

**Ожидаемый результат:**  
Ошибка: «Неверный месяц».

**Фактический результат:**  
Ошибка отсутствует.

**Окружение:**  
Windows 10, Chrome, Java 11.

## Дефект 4

**ID:** BUG-004  
**Название:** Поле «Год» принимает просроченный год  
**Описание:** При вводе года, который уже в прошлом, операция не блокируется.

**Шаги воспроизведения:**
1. Ввести год, который меньше текущего.
2. Заполнить остальные поля валидными данными.
3. Нажать «Продолжить».

**Ожидаемый результат:**  
Ошибка: «Карта просрочена».

**Фактический результат:**  
Операция проходит, ошибка не отображается.

**Окружение:**  
Windows 11, Chrome, Java 11.

---

## Дефект 5

**ID:** BUG-005  
**Название:** Отсутствует сообщение об ошибке при пустом поле «Владелец»  
**Описание:** При отправке формы с пустым полем «Владелец» сообщение об ошибке не отображается.

**Шаги воспроизведения:**
1. Открыть форму оплаты.
2. Оставить поле «Владелец» пустым.
3. Заполнить остальные поля валидными данными.
4. Нажать «Продолжить».

**Ожидаемый результат:**  
Отображается сообщение об ошибке: «Поле обязательно для заполнения».

**Фактический результат:**  
Сообщение об ошибке отсутствует, форма обрабатывается.

**Окружение:**  
Windows 10, Chrome, Java 11.

## Итог

Все 5 дефектов задокументированы и могут быть использованы для дальнейшего исправления и регрессионного тестирования.
