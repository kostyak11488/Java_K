# План автоматизации тестирования

## 1. Перечень автоматизируемых сценариев

Автоматизация будет выполнена для формы **оплаты по дебетовой карте**.

### Позитивные сценарии
1. Успешная покупка тура с использованием карты `APPROVED`
   - отображается сообщение об успешной оплате;
   - в базе данных сохраняется статус `APPROVED`.

### Негативные сценарии
2. Отказ в покупке тура с использованием карты `DECLINED`
   - отображается сообщение об ошибке;
   - в базе данных сохраняется статус `DECLINED`.

3. Отправка формы с пустыми обязательными полями
   - отображаются сообщения валидации.

4. Ввод некорректных данных карты:
   - неверный формат номера карты;
   - некорректный месяц;
   - прошедший год;
   - некорректный CVC.

## 2. Перечень используемых инструментов с обоснованием выбора

| Инструмент | Обоснование |
|----------|-------------|
| Java 11 | Требуется условиями проекта |
| Gradle | Удобное управление зависимостями |
| JUnit 5 | Фреймворк для написания автотестов |
| Selenide | Упрощает написание UI-тестов |
| Allure | Формирование наглядных отчётов |
| JDBC | Проверка данных, сохранённых в БД |
| PostgreSQL | Поддерживается приложением |
| Docker | Быстрое развёртывание СУБД |

---

## 3. Возможные риски при автоматизации

| Риск | Описание |
|----|---------|
| Нестабильность UI | Возможны падения тестов из-за асинхронной загрузки элементов |
| Общий банковский симулятор | В логах присутствуют запросы других студентов |
| Ошибки окружения | Проблемы с подключением к БД или сервисам |
| Ограниченные сроки | Необходимость выполнять проект в сжатые сроки |

---

## 4. Интервальная оценка с учётом рисков

| Вид работ | Время (часы) |
|---------|--------------|
| Анализ требований и планирование | 2–3 |
| Ручное тестирование | 3–4 |
| Разработка автотестов | 8–12 |
| Настройка отчётов | 2–3 |
| Подготовка отчётных документов | 2–3 |
| **Итого** | **17–25 часов** |

---

## 5. План сдачи работ

1. Подготовка плана автоматизации — 1–2 день  
2. Проведение ручного тестирования и оформление issue — 3–4 день  
3. Реализация и прогон автотестов — 5–7 день  
4. Подготовка отчёта по автоматизации и публикация результатов — 8 день  

Результатом выполнения работ являются:
- автотесты UI;
- проверки данных в базе данных;
- отчёт Allure;
- оформленные баг-репорты.
